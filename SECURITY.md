# Security Policy

## Supported Versions

LongLink is in early development. Security fixes are applied to the latest released version and the default branch.

## Reporting a Vulnerability

Do not open public issues for suspected vulnerabilities.

Report security issues through one of these channels:

- GitHub private vulnerability reporting
- Email `info@longlink.dev` with the subject `LongLink security report`

Include the affected version or commit, reproduction steps, impact, and any relevant logs or proof-of-concept details.

## Disclosure

We will confirm receipt, assess the report, prepare a fix when needed, and coordinate disclosure after affected users have a reasonable upgrade path.
